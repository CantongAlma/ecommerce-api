 STANDARDS FOLLOWED
All codes are properly commented following the requirements below:
  Javadoc comments added for every public class, method and interface
  Included @param, @return, @throws, @author and @see tags
  Added inline comments only for complex logic
  Avoided obvious comments to keep code clean



 1. JAVADOC FOR CLASSES
Format:
/**
* [Description of what the class does and its purpose]
* 
* @author [Your Full Name]
* @see [Related Class Name]
*/
